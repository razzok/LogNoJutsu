## Spacing Scale

Declared values (must be multiples of 4). Source: measured from existing index.html CSS rules.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside badges, tight inline spacing |
| sm | 8px | Compact element spacing, table cell padding-block |
| md | 12px | Card inner padding-block, form group gap, nav button padding |
| lg | 16px | Default section spacing, form row gap, card margin-bottom |
| xl | 20px | Card padding (existing `.card { padding: 20px }`) |
| 2xl | 24px | Page content padding, header padding |
| 3xl | 32px | Reserved for layout gaps — not used in this phase |

Exceptions:
- Modal overlay uses `position: fixed; inset: 0` — no spacing token applies to overlay itself
- Modal content box: `padding: 24px` (2xl), `max-width: 480px`, centered via flex
- Touch targets: not applicable (desktop-only tool used by security consultants)
- Badge inline padding: `1px 7px` — inherited from existing `.tier-badge` production CSS; not modified in this phase

## Component Inventory

New UI components introduced in this phase:

### 1. Scan Confirmation Modal

Triggered: when a campaign contains at least one technique with `requires_confirmation: true` and the user starts the simulation. Modal appears before execution begins. Scan does not proceed until "Confirm" is clicked.

**Structure:**
```
[overlay: position:fixed; inset:0; background:rgba(0,0,0,0.6); z-index:1000]
  [modal box: background:var(--bg2); border:1px solid var(--border); border-radius:8px;
              padding:24px; max-width:480px; width:90%; margin:auto;
              position:fixed; top:50%; left:50%; transform:translate(-50%,-50%)]
    [h2: "Network Scan Confirmation" — 15px, weight 600]
    [warning-line: ⚠ icon + "This will perform active network reconnaissance." — 13px, --orange]
    [info-list: 4 items, each 13px, --text, 8px gap between items]
      1. Target subnet: <auto-detected /24> (e.g. "192.168.1.0/24")
      2. Rate limit: 50 connections/second (configurable default)
      3. IDS/IPS notice: "Active scanning may trigger IDS/IPS alerts on monitored networks."
      4. Techniques: comma-separated list of technique IDs with requires_confirmation flag
    [button row: gap 8px, justify-content:flex-end, margin-top:20px]
      [Cancel Scan — .btn .btn-secondary]
      [Confirm Scan — .btn .btn-primary]
```

**Behavior:**
- Canceling closes the modal and aborts the simulation start (engine goroutine never launched)
- Confirming closes the modal and proceeds to simulation
- Modal is not dismissible by clicking the overlay or pressing Escape (deliberate — this is a safety checkpoint)
- Modal appears once per simulation start, not per technique

## Copywriting Contract

| Element | Copy |
|---------|------|
| Modal title | "Network Scan Confirmation" |
| Modal warning line | "This will perform active network reconnaissance on your local subnet." |
| Subnet info label | "Target subnet:" |
| Rate limit line | "Rate limit: 50 connections/second" |
| IDS warning line | "Active scanning may trigger IDS/IPS alerts on monitored networks." |
| Techniques label | "Scan techniques:" |
| Confirm CTA | "Confirm Scan" |
| Cancel action | "Cancel Scan" |
| AMSI badge text | "AMSI Blocked" |
| Elevation badge text | "Elev. Required" |
| Empty state (no scan techniques in campaign) | Modal does not appear — no empty state needed |
| Error state (scan confirmation API fails) | "Scan confirmation failed. Check browser console." — 13px, --red, inline below modal |
| Elevation-skipped technique row tooltip (optional) | "This technique requires administrator privileges. Run LogNoJutsu as admin to execute it." |

**Copywriting notes:**
- "Confirm Scan" / "Cancel Scan" are a symmetric verb+noun pair — specificity reduces accidental confirmation and the pair reads as a deliberate choice between two named actions
- "Elev. Required" abbreviated to fit badge width; full phrase "Elevation Required" used in HTML report
- No destructive actions in this phase — no destructive confirmation copy needed
- Report summary line follows D-06 pattern: "45/58 passed, 8 skipped (elevation required), 5 failed" — phrasing set by backend, not UI
