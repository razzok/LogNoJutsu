## Component Inventory

### New: Tier Badge (web UI)

Used in: technique table in `index.html`

```css
/* Add to existing style block */
.tier-badge { border-radius: 4px; padding: 1px 7px; font-size: 11px; font-weight: 600; display: inline-block; }
.tier1 { background: rgba(63,185,80,0.15); color: var(--green); border: 1px solid var(--green); }
.tier2 { background: rgba(210,153,34,0.15); color: var(--orange); border: 1px solid var(--orange); }
.tier3 { background: rgba(139,148,158,0.15); color: var(--muted); border: 1px solid var(--muted); }
```

Render logic (JS template string):
```javascript
t.tier === 1 ? '<span class="tier-badge tier1">T1</span>'
: t.tier === 2 ? '<span class="tier-badge tier2">T2</span>'
: t.tier === 3 ? '<span class="tier-badge tier3">T3</span>'
: '<span style="color:var(--muted)">—</span>'
```

**Source:** Specified in RESEARCH.md Pattern 6. Mirrors existing `.cs-badge` / `.ms-badge` conventions from reporter.

### New: Tier Column in HTML Report

```css
/* Add to existing report style block */
.tier1-badge { background: rgba(63,185,80,0.15); color: #3fb950; border: 1px solid #3fb950; border-radius: 4px; padding: 1px 7px; font-size: 11px; font-weight: 600; display: inline-block; }
```

**Source:** RESEARCH.md Pattern 5, citing existing `HasCrowdStrike` conditional column as precedent.

## Interaction Contract

### Tier Badge — Technique Table (web UI)

- Rendered as a non-interactive display badge. No click, no tooltip initially.
- Badge renders immediately when the technique list loads from `/api/techniques`.
- `t.tier === 0` (unclassified, only valid during audit window) renders as `—` — not as a badge.
- Column sorts are not required for Phase 14.

### Technique Table — Column Addition

- Add `<th>Tier</th>` to header row.
- Add tier badge `<td>` to each row in the `sorted.map(t => ...)` template string.
- Update `colspan` from 7 to 8 on any existing colspan references in the Techniques section.
- Do NOT add tier to the campaign card or campaign step display — tier lives on Technique only (RESEARCH.md anti-patterns).

### HTML Report — Tier Column

- Tier column appears conditionally via `HasTier` bool: only when at least one result carries `Tier > 0`.
- Badge renders inline in the results table, same row as technique name and pass/fail status.
- No new interactive elements in the report (static HTML, no JS).

### No New Pages, No New Routes

Phase 14 introduces no new navigation tabs, no new API routes exposed to the UI. The existing `/api/techniques` response already serializes the full `Technique` struct — once `Tier int` is added to the struct and populated in YAML files, the field becomes available automatically. No frontend route change needed.
