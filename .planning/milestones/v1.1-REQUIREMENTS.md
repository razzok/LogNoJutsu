## v1.1 Requirements

### Bug Fixes

- [x] **BUG-01**: Windows Audit Policy subcategories use locale-independent GUIDs instead of English names — fixes failure on non-English (German) Windows installations
  - *Outcome: Validated — 11 GUIDs replace 12 English names; preparation_test.go TestPoliciesUseGUIDs passes*
- [x] **BUG-02**: Preparation step failure messages include human-readable subcategory description alongside raw error detail (not just "exit status 87")

### Version

- [x] **VER-01**: Version string declared as injectable Go `var` in main.go, overridable at build time via `-ldflags "-X main.version=v1.1.0"` — defaults to `"dev"` when built without ldflags
  - *Outcome: Validated — `var version = "dev"` in cmd/lognojutsu/main.go; ldflags build verified*
- [x] **VER-02**: Server exposes `GET /api/info` endpoint returning `{"version": "..."}` JSON
  - *Outcome: Validated — TestHandleInfo_returnsVersion, TestHandleInfo_noAuthRequired, TestRegisterRoutes_infoNoAuth all pass*
- [x] **VER-03**: Web UI version badge fetches version from `/api/info` on page load — no more hardcoded `v0.1.0`
  - *Outcome: Validated — loadVersionBadge() in bare init block; human browser verification confirmed*

### UI Polish

- [x] **UI-01**: All German strings in Web UI replaced with English equivalents (Scheduler tab, PoC mode configuration)
  - *Outcome: Validated — grep for German indicator patterns returns 0 matches in index.html; "Kampagne laden" → "Load Campaign" caught and fixed post-commit*
- [x] **UI-02**: Preparation tab uses inline styled error panels instead of browser `alert()` for step failures
  - *Outcome: Validated — all 7 alert() calls replaced; showPrepError() and showInlineError/Success() helpers implemented*
- [x] **UI-03**: Dashboard displays total technique library count loaded from `/api/techniques` (currently 57)
  - *Outcome: Satisfied (with tech debt) — loadTechniqueCount() wired to /api/techniques; silently degrades when server uses --password*
- [x] **UI-04**: Tactic badges render correct colours for `command-and-control` and `ueba-scenario` tactics (currently grey due to missing funcMap entries)
  - *Outcome: Validated — TestTacticColor 3/3 subtests pass; red #f85149 and purple #bc8cff entries added to funcMap*
