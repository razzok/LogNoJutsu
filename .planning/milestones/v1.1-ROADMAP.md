## Phase Details

### Phase 8: Backend Correctness
**Goal**: Fix locale-dependent Windows Audit Policy failure (non-English Windows) and add build-time version injection with a public `/api/info` endpoint.
**Depends on**: Phase 7 (Nyquist validation baseline)
**Requirements**: BUG-01, BUG-02, VER-01, VER-02
**Success Criteria** (what must be TRUE):
  1. `ConfigureAuditPolicy` uses 11 stable Microsoft GUIDs — no English subcategory names
  2. Preparation step failure messages show human-readable description, not raw exit codes
  3. `var version = "dev"` in main.go; overridable via `-ldflags "-X main.version=v1.1.0"`
  4. `GET /api/info` returns `{"version":"..."}` without authentication
  5. All tests pass; no regressions
**Plans:** 2/2 plans complete

Plans:
- [x] 08-01-PLAN.md — GUID-based audit policy migration (BUG-01, BUG-02); 3 new preparation tests
- [x] 08-02-PLAN.md — Build-time version var + /api/info endpoint (VER-01, VER-02); 3 new server tests

#### Key Decisions
- `auditPolicies` extracted to package-level var for test access (TDD requirement)
- 12 English names replaced with 11 GUIDs — "Other Object Access Events" and "Scheduled Task" share GUID 0CCE9227
- Error format: `"<description>: failed (exit status N)"` — description field used, not raw GUID
- `/api/info` registered without authMiddleware — version is not sensitive, badge must load before login
- `testServer` updated with `Version: "test-v0.0.0"` — keeps existing tests valid

#### Tech Debt Carried Forward
- Two `// VERIFY` audit GUIDs (Audit Policy Change, Scheduled Task) need on-machine validation on non-English Windows

### Phase 9: UI Polish
**Goal**: Fix tactic badge colours for missing tactics, wire the version badge to `/api/info`, replace all `alert()` calls with inline panels, and translate all remaining German strings in the Web UI.
**Depends on**: Phase 8 Plan 02 (`/api/info` endpoint)
**Requirements**: UI-01, UI-02, UI-03, UI-04, VER-03
**Success Criteria** (what must be TRUE):
  1. `command-and-control` tactic badge renders red (#f85149); `ueba-scenario` renders purple (#bc8cff)
  2. Version badge `.version-badge` fetches from `/api/info` on page load
  3. Dashboard "Techniques Available" stat box shows count from `/api/techniques`
  4. All 7 `alert()` calls replaced with inline error/success panels
  5. Zero German strings in index.html (static HTML + JavaScript-generated text)
**Plans:** 3/3 plans complete

Plans:
- [x] 09-01-PLAN.md — Tactic badge colors (UI-04); TDD: TestTacticColor 3/3 subtests
- [x] 09-02-PLAN.md — Version badge + Techniques Available stat box (VER-03, UI-03)
- [x] 09-03-PLAN.md — English UI + inline feedback panels (UI-01, UI-02)

#### Key Decisions
- `tacticColor` remains inline closure in funcMap — consistent with existing pattern
- `confirm()` for deleteUser preserved — destructive action safeguard, not a disruptive alert
- `showInlineSuccess` auto-dismisses after 5 seconds; `showInlineError` persists until next action
- `loadTechniqueCount()` calls `/api/techniques` directly on init (not shared with loadScheduler)
- One missed German string (`Kampagne laden`) caught post-commit and fixed in separate commit

#### Tech Debt Carried Forward
- `/api/techniques` behind authMiddleware — "Techniques Available" stays `—` when server uses `--password`
- German strings remain in `reporter.go` htmlTemplate const (~15 strings) — UI-01 scoped to index.html only
- `09-VALIDATION.md` Nyquist flags (`nyquist_compliant`, `wave_0_complete`) remain false despite TestTacticColor passing
