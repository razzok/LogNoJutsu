    Per D-09: Wrap technique execution in RunWithCleanup so cleanup runs even if the technique body panics or context is cancelled.

    Replace the current `RunWithCleanup` function (lines 30-40 of executor.go) with this defer-style version:

    ```go
    // RunWithCleanup executes a technique (optionally as another user) and runs cleanup.
    // Cleanup is registered via defer — it fires even if runInternal panics.
    func RunWithCleanup(t *playbooks.Technique, profile *userstore.UserProfile, password string) (result playbooks.ExecutionResult) {
        // Register cleanup as deferred action — fires even on panic
        if strings.TrimSpace(t.Cleanup) != "" {
            defer func() {
                simlog.TechCleanup(t.ID, t.Cleanup, false)
                // Cleanup always runs as the launching user (we own the artifacts)
                _, _, cleanErr := runCommand(t.Executor.Type, t.Cleanup)
                result.CleanupRun = true
                simlog.TechCleanup(t.ID, "(cleanup completed)", cleanErr == nil)
            }()
        }
        result = runInternal(t, profile, password)
        return result
    }
    ```

    CRITICAL: The return variable MUST be named `result` (not anonymous). This is required for the deferred closure to write `result.CleanupRun = true` to the caller's copy. An anonymous return would cause the defer to mutate a local copy that the caller never receives. See RESEARCH.md Pitfall 1.

    Also add `result.Tier = t.Tier` in `runInternal` after the result initialization block (after line 61 in current code, after `TacticID: t.Tactic`). This populates the Tier field on ExecutionResult so the HTML report can access it:

    ```go
    result := playbooks.ExecutionResult{
        TechniqueID:   t.ID,
        TechniqueName: t.Name,
        TacticID:      t.Tactic,
        StartTime:     start.Format(time.RFC3339),
        Tier:          t.Tier,
    }
    ```
