## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between day cells in the calendar strip; icon-text gaps |
| sm | 8px | Accordion row bottom margin; result item spacing |
| md | 16px | Main content padding; card margin-bottom; form row gap |

| 3xl | 64px | Not used in Phase 12 |

Exceptions:
- Day cell touch target: 32×32px (not 44px — desktop-only internal tool, no mobile requirement)
- Digest row header padding: 10px top/bottom, 14px left/right (inherits from `.result-header` pattern — source: index.html line 89)
- Digest row body padding: 12px top/bottom, 14px left/right (inherits from `.result-body` pattern — source: index.html line 90)
- Day group label: 10px font-size (below scale — established pattern for meta labels, matches `.lbl` at 11px)
- Calendar strip bottom padding: 4px (overflow-x scroll clearance)

Accent reserved for:
1. Active day cell border (`border-color: var(--accent)`) and background tint (`rgba(88,166,255,0.3)`)

3. Phase 1 group label color (matches existing `.phase-poc_phase1` color: `var(--accent)`)
4. Clickable day cell cursor and hover state in calendar strip
5. Phase label for the Phase 1 day group header text

Additional semantic colors in scope (pre-existing, not accent):
- `--green: #3fb950` — complete day cell background tint + border; pass count text; "Done" status label
- `--muted: #8b949e` — pending day cells; gap day cells; collapsed row metadata; phase group label text; time window text
- `--red: #f85149` — Phase 2 day group label text (matches `.phase-poc_phase2`); fail count text
- `--border: #30363d` — default cell borders; accordion borders; digest row separators
- `--bg3: #21262d` — accordion header background; pending day cell fill

### Daily Digest Panel

Element IDs for calendar→digest link (D-11):
- Day row container: `id="digest-day-{N}"` where N is day number
- Digest body: `id="digest-body-{N}"`
- Chevron: `id="digest-chevron-{N}"`

### Accordion behavior (Claude's Discretion — resolved)

Use custom JS toggle class pattern (`.open` on `.digest-body` and `.digest-chevron`),
mirroring the existing `.result-item` / `.result-body.open` accordion pattern (index.html
lines 88–94). Do NOT use `<details>/<summary>` — programmatic open/close required for D-04
(auto-expand active day) and D-11 (calendar click → digest expand).

### Cell overflow handling (Claude's Discretion — resolved)

Single scrollable row with `overflow-x:auto` on `.day-strip`. For typical PoC schedules
(10–30 days), cells fit without wrapping. No CSS `flex-wrap` — wrapping breaks phase group
visual coherence.

### Panel visibility detection (Claude's Discretion — resolved)

Show both panels when `days.length > 0` (array is source of truth per D-02). Hide when
`days.length === 0`. Do NOT gate on `isPocRunning` — panels must persist after completion.

### Tooltip format (Claude's Discretion — resolved)

`title` attribute on each `.day-cell`:
Format: `"Day {N}: {technique_count} techniques | {pass_count} pass / {fail_count} fail"`
Example: `"Day 3: 5 techniques | 3 pass / 1 fail"`
Empty for pending days (no data yet): `"Day {N}: pending"`

## Interaction Contract

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Expand/collapse digest row | Click `.digest-header` | Toggle `.open` on `.digest-body` and `.digest-chevron` for that day |
| Auto-expand active day | On each render cycle | Call `openDigestRow(dayNum)` for day with `status === 'active'` after rebuilding HTML; re-apply previously expanded rows |
| Calendar → digest link | Click `.day-cell.active` or `.day-cell.complete` | Call `focusDayInDigest(dayNum)`: guard null, `openDigestRow(dayNum)`, `scrollIntoView({ behavior:'smooth', block:'nearest' })` |
| Pending cell click | Click `.day-cell.pending` or `.day-cell.gap` | No-op — non-interactive per D-11 |
| Status transition animation | Background-color change on data refresh | CSS `transition: background-color 0.4s ease` on `.day-cell` — no JS |
| Panel visibility | Days array changes | `days.length > 0` → `style.display = 'block'`; `days.length === 0` → `style.display = 'none'` |
| Stale-open-state preservation | Polling re-render | Before `innerHTML` rebuild: collect expanded day numbers from `.digest-body.open`; after rebuild: re-apply `openDigestRow()` for each; always apply active-day auto-expand on top |

## Polling Integration Contract

Source: RESEARCH.md Architecture Patterns — Pattern 3.

- Hook: Inside existing `pollStatus()` function at line 683 of index.html
- Condition: Only fetch `/api/poc/days` when `isPocPhaseActive(s.phase) || hasDayData`
- Module-level state: `let hasDayData = false;` — set to `true` on first non-empty array; reset to `false` when array returns `[]`
- New functions: `updateDayPanels(days)` called with result of `/api/poc/days` fetch
- Sub-functions: `renderDayCalendar(days)`, `renderDayDigest(days)` — follow `renderTimeline()` innerHTML pattern (Pattern 4)
