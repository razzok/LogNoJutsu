**1. TestPoCStop_DuringDayWait (D-03)**

Stop signal during `nextOccurrenceOfHour` sleep — the most common real-world scenario.

Create a `stopOnNthClock` helper that fires immediately for the first N-1 `After()` calls and blocks on the Nth call:

```go
type stopOnNthClock struct {
    fakeClock
    blockAt   int
    blockCh   chan time.Time
    callCount int
    mu        sync.Mutex
}

func (c *stopOnNthClock) After(d time.Duration) <-chan time.Time {
    c.mu.Lock()
    c.callCount++
    n := c.callCount
    c.mu.Unlock()
    if n >= c.blockAt {
        return c.blockCh
    }
    return c.fakeClock.After(d)
}
```

Setup: 2+1+1 = 4 days. Set `blockAt=2` (first After = day 1 scheduling wait fires immediately, second After = day 2 scheduling wait blocks). Start engine. `waitForPhase(eng, PhasePoCPhase1, 2s)`. Sleep 50ms for engine to enter second After(). Call `eng.Stop()`. Assert `waitForPhase(eng, PhaseAborted, 2s)`. Assert `eng.GetStatus().PoCDay == 1` (stopped during day 2 wait, so PoCDay should reflect day 2 was set but not completed). Assert engine did NOT reach PhaseDone.
