**Change 2: Add two async functions before the init block (before line 1315)**

Insert these two functions in the JS section, just before the `// Init` comment at line 1315:

```javascript
async function loadVersionBadge() {
  try {
    const r = await api('/api/info');
    if (r && r.version) {
      document.querySelector('.version-badge').textContent = r.version;
    }
  } catch (_) {}
}

async function loadTechniqueCount() {
  try {
    const techniques = await api('/api/techniques');
    if (Array.isArray(techniques)) {
      const el = document.getElementById('dashAvailable');
      if (el) el.textContent = techniques.length;
    }
  } catch (_) {}
}
```

Key points:
- Use the existing `api()` helper (same pattern as quickStart at line 809)
- `loadVersionBadge`: per D-08, if fetch fails leave badge as-is (the catch block swallows the error)
- `loadTechniqueCount`: per D-07, call `/api/techniques` directly (do NOT share from loadScheduler — that is called lazily on tab switch)
- Wrap both in try/catch for graceful degradation
